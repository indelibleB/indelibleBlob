# New Agent Onboarding Guide

**Version:** 2.0 (Updated March 5, 2026)

## Day 1: Foundation

1. Read `docs/foundation/CONSTITUTION.md` — Mission, values, and organizational structure
2. Read `docs/architecture/ARCHITECTURE.md` — The Sovereign Stack technical blueprint
3. Read `docs/foundation/AGENT_PLAYBOOKS.md` — Your specific role and responsibilities
4. Read `docs/coordination/SPRINT_STATUS.md` — Current sprint state (always read this first when starting a session)

## Day 2: Context

1. Review `docs/architecture/DECISIONS.md` — Decision history and rationale
2. Read `docs/agents/protocols/COMMUNICATION_PROTOCOLS.md` — How we communicate
3. Read `docs/agents/protocols/GIT_WORKFLOW.md` — Branching, commits, and review process
4. Explore the codebase — focus on your domain (`mobile/`, `website/`, `contracts/`, `shared/`)

## Day 3: Alignment

1. Set up your development environment (`docs/agents/protocols/DEVELOPMENT_SETUP.md`)
2. Read the current weekly coordination file in `docs/coordination/`
3. Review your task file in `docs/agents/tasks/` and begin daily tracking
4. Commit your first contribution

## Key Directories

| Directory | Purpose |
|-----------|---------|
| `docs/foundation/` | Identity — who we are |
| `docs/architecture/` | Blueprint — what we build |
| `docs/agents/protocols/` | Operations — how we work |
| `docs/agents/tasks/` | Your daily task tracking |
| `docs/coordination/` | Sprint status and weekly archives |
| `docs/security/` | Threat models, audit logs, key management |
| `docs/references/` | External SDK documentation |

## Checklist

- [ ] Values alignment confirmed (read Constitution)
- [ ] Role and responsibilities understood (read Playbooks)
- [ ] Development environment working
- [ ] Repository access verified
- [ ] Task file initialized in `docs/agents/tasks/`
- [ ] First contribution merged
